# LLM-MBTI Recommender — Spec

## 개요

사용자가 MBTI 유형·IT 직무·주된 작업·중요한 가치를 차례로 입력하면, 그 조합에
맞춰 사전 구축된 추천 매핑(affinity 데이터 + 템플릿)으로부터 추천 모델(메인 1
+ 대안 1-2)과 변수 치환된 시스템 프롬프트가 즉시 표시된다. 같은 입력은 항상
같은 결과를 만든다.

## 범위

### 포함
- 4단계 입력 흐름: ① MBTI 16유형 단일 선택 → ② IT 직무 다중 선택 → ③ 주된 작업
  다중 선택 → ④ 우선순위 다중 선택
- 입력값을 쿼리스트링으로 보존한 결과 페이지 URL
- 추천 결과: 메인 모델 1개 + 대안 1-2개 (정적 매핑 룩업으로 결정적)
- 메인 카드 구성: 모델명, 추천 근거 텍스트(템플릿 + 변수 치환), 사용자 맞춤
  시스템 프롬프트(템플릿 + 변수 치환, 복사 버튼)
- 대안 카드 구성: 모델명, 짧은 강점·왜 대안인지 한 줄
- 공유 액션: URL 클립보드 복사 + 소셜 미리보기용 OG 이미지
- 결과 페이지에서 "다시 진단하기"로 1단계로 돌아가기

### 제외
- **런타임 LLM API 호출** — 추천은 정적 룩업으로 결정. 런타임에 외부 LLM에
  의존하지 않는다(비용·지연·실패 가능성 제거).
- **로딩 상태·실패 재시도 화면** — 룩업이 결정적이라 별도 화면 불필요.
- **로그인·계정·DB 사용자 식별** — 익명 게스트만. 본인 학습·포트폴리오 단계라
  사용자 식별이 학습 신호에 기여하지 않음.
- **실제 LLM API 호출 라우팅·실행 기능** — 추천만, 실행은 사용자가 외부에서.
  범위 폭증 방지.
- **MBTI 모르는 사용자용 간이 quiz** — `idea.md`의 "Might Be True". 사용 데이터
  쌓인 뒤 검증.
- **결과 카드 이미지 다운로드, Web Share API** — URL + OG로 충분한지 먼저 본다.
- **다국어 UI** — 한국어 우선. 영어는 MVP 이후.
- **유료화·결제·affiliate 링크** — 학습 신호와 무관.
- **팀·조직 단위 진단 보고서** — B2B 흐름은 검증 사이클 길어 포트폴리오 목적과
  충돌.
- **외부 모델 사이트 바로가기 링크** — 사용자가 결과 카드에서 명시적으로
  요청하지 않음.
- **모델 메타데이터(컨텍스트 길이·속도·비용) 비교 표시** — 사용자가 결과 카드
  구성에서 선택하지 않음.

## 시나리오

### 1. 4단계 진단 happy path

- **Given** — 사용자가 진단 시작 페이지에 진입했고, 어떤 단계도 입력하지 않았다
- **When** — MBTI 16유형 그리드에서 "INTJ" 카드를 클릭하고 "다음"을 누른 뒤,
  직무 화면에서 "인공지능"과 "어플리케이션SW"를 체크하고 "다음", 작업 화면에서
  "코드 작성·리뷰"와 "기술 리서치·학습"을 체크하고 "다음", 우선순위 화면에서
  "정확성"과 "창의성"을 체크하고 "결과 보기"를 누른다
- **Then** — 결과 페이지로 이동하고, 메인 추천 카드 1개 + 대안 카드 1-2개가
  표시된다. 메인 카드에 모델명·추천 근거 텍스트·시스템 프롬프트(복사 버튼)가
  보인다.

성공 기준:
- [ ] 1단계에서 MBTI 카드 미선택 시 "다음" 버튼이 비활성 상태로 보인다
- [ ] 2-4단계에서 다중 선택 0개일 때 "다음"/"결과 보기" 버튼이 비활성 상태로 보인다
- [ ] 4단계 "결과 보기" 클릭 후 결과 페이지 URL에 모든 입력값이 쿼리스트링으로
      포함된다 (예: `?mbti=INTJ&jobs=ai,app&tasks=code,research&priorities=accuracy,creativity`)
- [ ] 결과 페이지에 "추천 모델: <모델명>" 형식의 메인 카드 1개가 보인다
- [ ] 메인 카드 아래에 1-2개의 대안 카드가 보이고, 각각 모델명과 한 줄 강점이 보인다
- [ ] 메인 카드에 추천 근거 텍스트가 보이고, 사용자의 MBTI·직무·작업·우선순위
      이름이 텍스트 안에 노출된다
- [ ] 메인 카드에 시스템 프롬프트 영역과 "복사" 버튼이 보이고, 프롬프트 안에
      사용자의 MBTI·직무·작업·우선순위 값이 변수 치환되어 들어 있다

### 2. 시스템 프롬프트 복사

- **Given** — 결과 페이지가 표시되어 있고 메인 카드의 시스템 프롬프트가 보인다
- **When** — 시스템 프롬프트 영역의 "복사" 버튼을 클릭한다
- **Then** — 클립보드에 프롬프트 텍스트가 복사되고, "복사됨" 같은 시각적 피드백이
  잠시 표시된다

성공 기준:
- [ ] "복사" 버튼 클릭 → 클립보드의 텍스트가 카드에 표시된 시스템 프롬프트와 일치한다
- [ ] 클릭 직후 버튼 라벨이 "복사됨"으로 잠시 바뀌고 자동으로 원복된다

### 3. 결과 URL 공유

- **Given** — 결과 페이지가 표시되어 있다
- **When** — "공유" 버튼을 클릭한다
- **Then** — 결과 페이지의 현재 URL이 클립보드에 복사되고, 복사 완료 피드백이
  잠시 표시된다. 그 URL을 외부 메신저에 붙여 넣으면 결과 카드 모양의 OG 이미지가
  미리보기로 보인다

성공 기준:
- [ ] "공유" 버튼 클릭 → 클립보드의 텍스트가 현재 결과 페이지 URL과 일치한다
- [ ] 결과 페이지 HTML에 OG 이미지·제목·설명 메타태그가 포함되어 있고, OG 이미지에
      메인 추천 모델명과 MBTI가 보인다

### 4. 공유 받은 URL 재방문 (결정적 매칭)

- **Given** — 사용자 A가 진단을 마치고 결과 페이지 URL을 받은 사용자 B가 있다
- **When** — 사용자 B가 그 URL을 새 브라우저 탭에서 연다
- **Then** — 사용자 A가 봤던 것과 동일한 메인 모델·대안·근거 텍스트·시스템 프롬프트가
  표시된다 (추천 매핑 데이터가 변경되지 않은 한)

성공 기준:
- [ ] 같은 입력 쿼리스트링 URL을 두 번 열면 두 페이지의 메인 모델명·시스템 프롬프트
      텍스트가 일치한다
- [ ] 같은 입력 쿼리스트링 URL을 두 번 열면 두 페이지의 추천 근거 텍스트가 일치한다

### 5. 잘못되거나 누락된 입력 URL로 결과 페이지 직접 진입

- **Given** — 사용자가 결과 페이지 URL을 직접 입력하거나 링크로 진입했다
- **When** — 쿼리스트링에 MBTI 값이 없거나 16유형 외 값이 들어 있거나, 직무·작업·
  우선순위 중 하나라도 알 수 없는 값이 들어 있다
- **Then** — 결과 페이지를 렌더링하지 않고 1단계(MBTI 입력) 화면으로 보낸다

성공 기준:
- [ ] `?mbti=` 빈 값 또는 누락 → 1단계 MBTI 화면이 표시된다
- [ ] `?mbti=ZZZZ` 같은 16유형 외 값 → 1단계 MBTI 화면이 표시된다
- [ ] `jobs`·`tasks`·`priorities` 중 어떤 값도 정의된 목록에 없을 경우 → 1단계
      MBTI 화면이 표시된다

### 6. 결과 페이지에서 다시 진단하기

- **Given** — 결과 페이지가 표시되어 있다
- **When** — "다시 진단하기" 버튼을 클릭한다
- **Then** — 1단계(MBTI 입력) 화면으로 이동하고 이전에 선택한 입력은 모두 비워진다

성공 기준:
- [ ] "다시 진단하기" 클릭 후 URL이 1단계 화면 경로로 바뀐다
- [ ] 1단계 화면의 MBTI 그리드에 아무 카드도 선택되어 있지 않다

## 불변 규칙

### 결정적 매칭
- 동일한 입력 조합(`mbti`+`jobs`+`tasks`+`priorities`)에 대해 추천 메인 모델명·
  대안 모델명·추천 근거 텍스트·시스템 프롬프트는 항상 동일하게 표시된다 (추천
  매핑 데이터가 변경되기 전까지). 입력 조합 항목들의 **순서**가 달라져도 결과는
  동일해야 한다.

### 프라이버시
- 입력값은 익명으로만 처리한다. 결과 페이지 URL 외에 사용자 식별 가능한 정보를
  저장하거나 표시하지 않는다.

### 추천 후보 제약
- 추천 결과(메인·대안 모두)는 사전에 정의된 후보 풀(Claude·GPT·Gemini 라인업의
  large·small 조합, 5개 내외)에서만 선택된다. 후보 풀 밖의 모델명은 결과에
  표시되지 않는다.

### 런타임 외부 의존성 제거
- 결과 페이지 렌더링은 외부 LLM API에 의존하지 않는다. 추천 매핑 데이터(JSON +
  텍스트 템플릿)는 빌드 산출물에 포함된다.

## 의존성

- 추천 매핑 데이터: MBTI·직무·작업·우선순위 각각의 모델 affinity 점수 + 추천 근거·
  시스템 프롬프트 텍스트 템플릿. 구체 데이터·생성 방식은 `plan.md`에서 결정.
- 추천 후보 모델 풀 정의: 구체 모델·버전 목록은 `plan.md`에서 확정.

## 미결정 항목

- 추천 매핑 데이터의 갱신 주기·운영 방식 — 모델 풀이 바뀌었을 때 어떻게 다시
  생성·배포할지는 운영 단계에서 결정한다.
- MBTI 모르는 사용자용 간이 quiz 진입점 — `idea.md`의 "Might Be True". MVP 출시
  후 사용 데이터 본 뒤 결정.
